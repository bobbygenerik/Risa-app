# AGENT RULES & CODING STANDARDS

**CRITICAL: ALL AGENTS WORKING ON THIS PROJECT MUST ADHERE STRICTLY TO THESE RULES.**

This document outlines the core principles, goals, and operational protocols for building the Risa-app.

## 1. Project Vision
*   **Goal**: Build an industry-leading IPTV application.
*   **Benchmark**: The app must rival major streaming platforms like **Disney+, Netflix, and HBO Max** in terms of quality, feel, and performance.
*   **Aesthetics**: The design must be premium, modern, and polished. "Simple" or "Basic" is unacceptable.

## 2. Core Priorities
*   **User Experience (UX)**: This is paramount. The app must be intuitive, fluid, and delightful to use.
*   **Efficiency**: The app must be highly optimized.
*   **Performance**: Load times must be **minimal**. Every millisecond counts.

## 3. Development Protocols
*   **Forward-Only Problem Solving**: 
    *   If a fix for a problem is unsuccessful, **reverting to a previous problematic state is NEVER the solution.**
    *   You must find a new, working solution. Do not backtrack to broken code.
*   **Resource Management**: 
    *   The app must operate with the smoothness and capability of major streaming apps, but without their massive server resources.
    *   Code must be efficient, lightweight, and optimized for limited resources.

## 4. Collaboration Roles
*   **User**: Novice Developer.
    *   Relies on the agent for expertise, best practices, and architectural guidance.
*   **Agent**: Expert Consultant & Lead Engineer.
    *   Provide clear, expert guidance.
    *   Explain complex concepts simply when necessary.
    *   Take initiative to ensure quality standards are met.

## 5. Strict Adherence
*   These rules are not suggestions. They are the **fundamental constraints** of the project.
*   Check every decision against these principles.
